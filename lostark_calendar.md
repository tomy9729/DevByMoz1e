# 로스트아크 특화 달력 서비스 프로젝트 계획서

## 1. 프로젝트 설명

### 1-1. 프로젝트 개요

로그인 없이 사용할 수 있는 로스트아크 특화 달력 서비스다.

모든 사용자는 로스트아크 관련 일정을 달력에서 확인할 수 있고, 카카오톡 연동자는 직접 일정 등록, 개인 알림, 봇 기능을 사용할 수 있다.

장기적으로는 카카오톡 단체방 기반 그룹 일정 관리와 공대 운영 자동화 기능까지 확장한다.

### 1-2. 핵심 컨셉

- 달력이 서비스의 중심이다.
- 알림은 일정에 연결되는 기능이다.
- 로스트아크 일정은 시스템 일정으로 제공한다.
- 사용자 일정과 로스트아크 일정을 하나의 달력에서 함께 보여준다.
- 카카오톡 연동자를 기준으로 일정 등록과 알림 기능을 제공한다.
- 로그인 기능은 만들지 않는다.
- 초기에는 로스트아크만 고려한다.
- 수익화는 후순위로 두며, 필요 시 스마일게이트 승인 여부를 먼저 확인한다.

### 1-3. 주요 사용자

| 사용자 유형   | 설명                                                           |
| ------------- | -------------------------------------------------------------- |
| 미연동 사용자 | 로아 달력을 조회하는 사용자                                    |
| 개인 연동자   | 카카오톡 1대1 연동 후 개인 일정/알림을 사용하는 사용자         |
| 그룹 사용자   | 카카오톡 단체방에 봇을 연동해 그룹 일정/알림을 사용하는 사용자 |

## 2. 개발 목표

### 2-1. 단계별 목표

| 단계 | 목표                                            |
| ---- | ----------------------------------------------- |
| 1차  | 비연동 사용자 대상 로아 달력 조회 서비스        |
| 2차  | 카카오톡 개인 연동자 대상 개인 일정/알림 서비스 |
| 3차  | 그룹 사용자 대상 단체 일정/공대 운영 서비스     |
| 4차  | 각 서비스 고도화                                |
| 최종 | 후원, 광고, 그룹 기능 중심 수익화 검토          |

### 2-2. 1차 배포 기준

- 미연동 사용자가 웹에서 로아 달력을 조회할 수 있다.
- 로아 시스템 일정이 달력에 표시된다.
- 일정 상세 정보를 확인할 수 있다.
- eventType별 색상이 적용된다.
- 표시 필터를 사용할 수 있다.
- 일정 등록/알림 기능은 카카오톡 연동 안내로 제한한다.

## 3. 서비스 정책

### 3-1. 접근 권한

| 기능                  | 미연동 사용자 | 개인 연동자 | 그룹 사용자 |
| --------------------- | ------------: | ----------: | ----------: |
| 로아 달력 조회        |             O |           O |           O |
| 로아 일정 상세 보기   |             O |           O |           O |
| 달력 필터             |             O |           O |           O |
| 일정 색상 표시        |             O |           O |           O |
| 사용자 일정 등록      |             X |           O |           O |
| 사용자 일정 수정/삭제 |             X |           O |           O |
| 로아 일정 알림 등록   |             X |           O |           O |
| 사용자 일정 알림 등록 |             X |           O |           O |
| 봇 명령어             |             X |           O |           O |
| 1대1 알림             |             X |           O |           X |
| 단체톡 알림           |             X |           X |           O |
| 그룹 일정 공유        |             X |           X |           O |

### 3-2. 일정 정책

- 달력에 표시되는 모든 것은 일정으로 취급한다.
- 사용자 일정과 로아 시스템 일정은 같은 달력에 표시한다.
- 로아 시스템 일정은 사용자마다 복사 저장하지 않는다.
- 로아 시스템 일정은 공통 데이터로 관리한다.
- 사용자별로는 표시 여부, 알림 여부, 색상 설정만 저장한다.
- object 구조는 이 문서에서 확정하지 않는다.
- 달력 라이브러리는 FullCalendar를 사용한다.
- FullCalendar의 Event Object, Event Source 구조를 참고한다.

### 3-3. 알림 정책

- 알림은 일정에 연결된다.
- 달력에 표시되는 모든 항목은 알림 등록 가능 대상이다.
- 미연동 사용자는 알림을 등록할 수 없다.
- 카카오톡 연동자만 알림을 등록할 수 있다.
- 서버가 최종 알림 메시지를 생성한다.
- 봇은 서버가 준 메시지를 그대로 발송한다.

### 6-1. 이 문서의 역할

- 프로젝트 설명 문서
- 기획 정리 문서
- 개발 방향 기준 문서

- 실제 DB object 구조를 확정하지 않는다.
- API 상세 스펙을 확정하지 않는다.
- 봇 명령어를 확정하지 않는다.
- 디자인 상세를 확정하지 않는다.
- 수익 모델을 확정하지 않는다.

### 6-3. Codex 작업 기준

- 작업 전 이 문서를 먼저 참고한다.
- 서비스 컨셉, 사용자 접근 권한, 일정/알림 정책은 이 문서를 기준으로 한다.
- object 구조는 실제 구현 시 FullCalendar 구조와 현재 코드 구조를 참고하여 별도 설계한다.
- 봇 명령어는 별도 봇 스펙 문서에서 관리한다.
